
	Все запрсы HTTP  не зависят друг от друга. кожадый запрос как новый пользователь.
	
	Куки делятся минимум на 2 типа. 
		Сессионая - она уничтожается после закрытия браузера. 
		Персистемные куки - хранятся на где-то на диске и имеют срок жизни.
		а также MAX-AGE=2592000
		domain=.hexlet.io; path=/; expires=Fri
		
	Не судествует заголовка для удаления  куки. Надо установить отрицательный MAX_AGE = -3, и она будет удалена сразу.

ТЕСТЫ
	HTTP — stateless protocol. Что это значит?
		каждая пара «HTTP-запрос + ответ» независимая, и протокол не «помнит» ничего и прошлых запросах
	
	Может ли заголовок "Set-cookie" встречаться несколько раз в одном ответе (response)?
		да
		
	Где у пользователя хранятся cookie, отправленные сервером?
		в браузере
		
	Пользователь зашел на сайт интернет-магазина (без авторизации, как гость), добавил несколько 
	 товаров в свою корзину, закрыл браузер, перезагрузил компьютер, открыл браузер, вернулся 
	 на сайт и увидел, что товары все еще находятся в его корзине. Какой тип cookie 
	 использовался (скорее всего) для достижения такого эффекта?
		персистентные куки (persistent cookies)
		
	В куки есть такой фрагмент: domain = .yandex.ru. Для каких адресов будет использоваться этот куки?
		для yandex.ru и всех поддоменов вида x.yandex.ru

ЗАДАНИЕ
	Используя telnet выполните запрос к hexlet.local (расположен на localhost) на порт 8080. 
	 Параметры запроса: глагол get, страница /account, протокол http 1.1, куки name 
	 со значением user и secret со значением secret_hash;
	Запишите ваш request в файл solution;
							// BEGIN
							GET /account HTTP/1.1
							Host: hexlet.local
							Cookie: name=user; secret=secret_hash
							// END
