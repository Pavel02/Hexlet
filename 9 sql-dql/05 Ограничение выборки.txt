
	
						select * from products limit 1;
						
	limit никак не регламентирует порядок, и при одном и том же запросе могут быть разные результаты. Поможет  order by
	
	offset  для того чтобы показывать не с первого результата
						select * from products limit 1 offset 2; 	// Будет пропущено 2 первых значения
	
	Не смотря на то, что пропускается вывод значений.  offset Все равно по ним проходится. Поэтому при большой базе может тормозить.

ТЕСТЫ
	Можно ли в limit (без использования других ключевых слов) управлять порядком вывода?
		нет
		
	В комбинации с каким ключевым словом чаще всего используют limit?
		order
		
	Как использование offset влияет на производительность?
		Пропущенные с помощью offset данные в любом случае проходят выборку, поэтому при большом значении 
		 offset запрос будет медленным
