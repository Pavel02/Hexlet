
	Особенности РНР
		Изначально не ЯП. просто набор скриптов для web разработки.
		Превратился в ЯП, но наследие осталось.
		Единственный ЯП в который встроена работа с web.
		
	Реалии 
		Можно создавать сайты не понимая как работает web.
		Отсутствует структура, что ведет к сложной поддержке.
		
	Фреймворки
		Платформа, определяющая способ построения приложения. Берет на себя решение общих задач.
			Диспетчеризация (Роутинг)
			Request/Response
			Шаблонизация
		
	О курсе
		Встроенные возможности PHP по работе с HTTP
		MVC архитетура
		Шаблонизация
		Построение микрофреймворка
