
	Цель: Познакомиться со способом запуска программ из под создавшего их пользователя
	
					ls -la /etc/shadow		// Посмотрим на права на файл shadow  где хранятся зашифрованные пароли
							выведет  -rw-rw---- 
							
					passwd					// для любого пользователя меняет пароль. производит запись в этот файл 
					
					type passwd				// информация о команде 
					ls -la /usr/bin/passwd 		// Посмотрим права на исполняемы файл		
файл ссылается на /bin/bussybox.suid	
					
					ls -la /bin/bussybox.suid  
		
	Обычно программа запускается из под пользователя и наследует именно его права (не важно кто создал программу).
	А вот когда к файлу добавляется .suid  то программа запускается с правами своего создания. Именно так многие программы 
	получают доступ к изменению файлов, которые запрещено менять текущему пользователю.
	 Именно поэтому будет работать программа passwd которая меняет файл shadow к которому нет 
	 прав на изменение текущего пользователя. 
	 
	 SGID  аналогично но для завпуска с правами группы
