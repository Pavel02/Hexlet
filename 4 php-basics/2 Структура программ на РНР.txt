
	Нельзя сразу писать код. только в тегах  <?php  ?>
	Раньше был HTML файл и в него встраивался код PHP.
		Но сейчас стандартом не приветсвуются множественные конструкции в одном файле.
		
	Если файл только РНР то закрывающий не ставим тег ?>
	
	Комментарии в РНР 
		// Однострочные
		
		/*
		* Multi Line
		* Comment
		*/
		
		# Почти не используется
		
		
	Statements - инструкции.
	Вся программа это набор инструкций.
	; это так называемая инструкция.
				;;; 		// 3 пустые инструкции. Это валидный код на РНР.
				
	Выражения.
	Выражения вычисляются, а инструкции выполняются! 
		" ";			// выражение пустая строка
		3;				// выражение 3 
		1 + 2;			// выражение 3
		$name = "value"; 	// выражение присаивание
		
		
	Списки инструкций.
		function name() {....}		// В конце нет ;  Это инстукция определение функции.
		if (...) {....}				// В конце нет ; 
	
		{}  РНР рассматривает фигурные скобки как   Statements List  , как список инструкций.
		
		Анонимная функция рассматривается как выражение и в ее конце ставим ;
			function ($a, $b) {
				return $a ** $b;
			};
		 
		
	Ответы на вопросы:	
	Крайне важно понимать для того чтобы знать в каких местах что вы можете использовать и как вы можете 
	 комбинировать элементы языка. Например вы не можете написать myFunc(echo 1), но можете myFunc(print_r(3)). 
	 Любой аргумент функции это выражение, что автоматически означает возможность на место аргумента ставить 
	 любую вычислимую штуку, например другую функцию: func(anotherFunc(3 + 4)).
	Инструкция выполняет действие, выражение вычисляет результат.
