
	БД место в котором содержатся данные вашего приложения 
	БД сама по себе статическая вещь, и она должна чемто управляться СУБД
	
		В простейшем случае БД может являться обычными файлами, но на практике 
		 не используют, так как это не оптимальный путь.
		 
	История
		К 1980 году уже появилось множество СУБД.
		У каждого свой собственный язык запросов. 
			БД существуют отдельно от програмного кода, но всеравно должен быть механизм управления ими.
		В 1986 году вышел первый стандарт SQL86.
			Общий язык запросов, чтобы увеличить переносимость БД.
		
	SQL это формальный декларативный язык, применяемый для создания, модификации и управления данными в произвольной 
	 реляционной БД, управляемой соответствующей СУБД. 
	
	SQL для реляционных БД.

	СУБД 
		PosgreSQL
		Sqlite (популярная БД, построена на файлах, является встраиваемой, для нее не нужна СУБД, SQL работает прямо в вашем коде)
		MySQL (MariaDB, Percona)
		MS-SQL
		Oracle
	
	На верхнем уровне все реляционные БД практически одинаковы. Все они поддерживают SQL стандарт.
	
	Изучение БД.
		Не пытайтесь учить конкретную СУБД. Учите SQL используя выбранную СУБД.
		
	Во всех БД имеются свои расширения и особенности, но SQL будет в них всех работать.
	
	
	SQL Parts
		1. Data Definition Language (DDL)
			Создание структуры БД, которое включает в себя таблицы и поля в них.
		2. Data Manipulation Language (DML) 
			Вставлять удалять изменять данные.
		3. Также теория что такое реляционные БД, что такое нормализация БД и т.д.
