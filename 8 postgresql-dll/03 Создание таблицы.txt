
	Когда мы вашли в СУБД нам надо подключиться к БД для работы 
					\с hexlet				// Подключение к БД с именем  Хекслет.
					
					\d						// Список таблиц в БД к которой мы подключились
					
					\d products				// Узнаем какие колонки есть в таблице  products (в подключенной БД  хекслет)
					
	Есть 2 подхода к именованию таблиц (в единственном и множественном числе) главное чтобы в пределах 
	 одной БД было единобразно.
	 
	Создаем таблицу в которой имена колонок и их типы.
					CREATE TABLE products (
						product_no integer,
						name character varying,
						description text,
						count integer,
						price numeric,
						created_at timestamp
					);
					
	Код создания таблиц может быть еще больше, и писать его в консоль неудобно, поэтому
	 приемлема практика записать этот код в файл (например  ddl.sql) и вызвать файл из консоли.
					psql -U postgres hexlet < ddl.sql		// Здесь указываем имя БД и перенаправляем поток в файл.
					
	В отличие от MySQL Postgresql является БД со стройгой типизацией. Никаких неяных преобразований типов быть не может.
	postgresql намного ближе к стандарту.
	 
	Типы Postgresql
		integer
		boolean
		character varying (по умолчанию строчка неограниченной длины (похоже на var-char в MySQL)
		text	(не входит в стандарт SQL, но почти во всех СУБД есть)
		numeric   (позволяет хранить дробные числа) (очень удобно для цен и нужно использовать точку)  float  округляет
		timestamp    (время с начала эпохи Линукс)
