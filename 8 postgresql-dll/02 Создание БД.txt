
	Будем работать с postgresql , но почти во всех БД будет работать без изменений.
	
	Создание БД всегда осуществляется из под какого-то пользователя, конкретно для PostgreSQL 
	 он должен обладать правами суперпользователя или иметь привелегию на создание БД  CREATEDB
	 
	 
	1. Мы должны в консоли (командной строке) подключиться к БД под определенным  юзером.
					psql -U postgres 			// в этом примере пользователь  postgres (в БД postgresql этот пользователь является суперпользователем) 
												// лучше обязательно использовать отдельно созданного пользователя 
												// потому что если приложение будет входить под этим пользователем, то жто не безопасно.
												
					CREATE DATABASE test2;		// Создание БД с именем  test2

					\? 							// Справка по командам
					
	Метаинформацию о БД тоже хранится в самой БД.			
					
					\l							// Посмотреть все имеющиеся БД (это управляющая конструкция.)
												// в реальности за место нее выполняется SQL запрос.
												
					
					DROP DATABASE test2;		// Удаление БД с именем  test2.
					
					\q							// Выход из СУБД
					
					
					
	В postgresql  имеются специальные шорткаты
					createdb -U postgres test	// Находясь вне СУБД. набрав в командной строке такой шорткат мы создадим БД с именем test 
					
					dropdb -U postgres test		// Находясь вне СУБД. набрав в командной строке такой шорткат мы удалим БД с именем test 
					
					ALTER DATABASE test RENAME TO test2; 	// Меняем имя БД
					
					ALTER DATABASE test OWNER TO vagrant; 	// Меняем владельца БД (хотим сделать пользователя vagrant)
					
					ALTER DATABASE test SET key value; 		// Изменение параметров (подробнее в документации)
